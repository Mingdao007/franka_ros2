# Validated Workflow (Fast Iteration)

## 1) Build
```bash
cd <your_workspace>
source /opt/ros/humble/setup.bash
colcon build --packages-select franka_example_controllers franka_gazebo_bringup --symlink-install
source install/setup.bash
```

## 2) Manual launch with RViz
```bash
cd <your_workspace>
source install/setup.bash
ros2 launch franka_gazebo_bringup gazebo_hybrid_circle_force.launch.py use_rviz:=true
```

Expected behavior:
1. Robot spawns in Gazebo, controller activates (~3s).
2. Descent phase: arm descends with high-stiffness impedance control.
3. Contact detected (|Fz| > 2N): switches to circle phase.
4. Circle phase: cosine soft-start ramp, then steady circle tracking + Z force PID.
5. RViz shows red (desired) and blue (measured) ink trails.

## 3) Run a short baseline experiment (batch)
```bash
cd <your_workspace>
source install/setup.bash
REPEATS=1 DURATION=20 KEY_RUN_POLICY=first-gui bash src/run_hybrid_circle_force_experiment.sh
```

Notes:
- `REPEATS=1` and short `DURATION` are preferred for quick tuning loops.
- Script performs activation polling and cleanup automatically.
- Expected activation time: ~3s.

## 4) Locate outputs
- Base folder: `src/results/hybrid_circle_force/<timestamp>/`
- Key files:
  - `report.md`
  - `summary_metrics.png`
  - `run_1/launch.log`
  - `run_1/collector.log`
  - `run_1/wrench.csv`
  - `run_1/internal.csv` (if controller internal log is available)
  - `run_1/pseudo_ink.png` (matplotlib-based ink trajectory plot)

## 5) Gate evaluation logic
- Pass condition requires all:
  - `|mean(Fz error)| < 0.300 N`
  - `Fz RMSE < 0.600 N`
  - `XY RMS < 0.005 m` (if available)

## 6) Metric standard (reporting policy)
- Do not report MSE.
- Always report RMSE.
- Also report normalized RMSE values when possible:
  - `NRMSE_target = RMSE / |F_target|`
  - `NRMSE_range = RMSE / (F_max - F_min)` (use `N/A` if denominator is zero)
- Keep raw-unit RMSE in N as the primary acceptance metric.

## 7) Minimal tuning loop
1. Run one baseline.
2. Identify first failed gate.
3. Change one group of parameters only.
4. Re-run short test.
5. Keep artifact paths in summary for traceability.

## 8) Ink trail visualization
- **RViz markers** (primary):
  - Blue `~/ink_trail`: measured EE position, only when contact force > threshold.
  - Red `~/ink_desired`: desired circle trajectory, always drawn during circle phase.
  - Launch with `use_rviz:=true` to enable.
  - Config: `hybrid_circle_force.rviz`.
- **Pseudo-ink PNG**: always generated by experiment script (matplotlib).
